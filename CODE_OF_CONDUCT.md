# Code of conduct

This is a short one, written for this project rather than copied from a template, because this project
has a specific and predictable failure mode.

## The one rule that matters here

**Disagreement about the framework is welcome. Argument about theology belongs somewhere else.**

This repository is about a *method* — a file format, a set of constraints on how an AI assistant is
used, and a validator. It is deliberately empty of findings and takes no doctrinal position.

That is not squeamishness. It is what makes the tool usable by people who disagree profoundly with each
other. Someone should be able to file a bug about the wikilink validator without either of us knowing or
caring what the other believes about baptism. The moment the issue tracker becomes a place to argue
about what a text means, it stops being useful to everyone who came here for the tooling.

So: no doctrinal advocacy in issues, pull requests, or commit messages. Not because the questions are
unimportant — they are the most important questions there are — but because this is not where they get
settled.

## The ordinary expectations

- Assume the other person is arguing in good faith, and argue with what they actually said.
- Criticise code, documentation and reasoning. Do not characterise people.
- No harassment, personal attacks, or abuse of any kind.
- If someone asks you to stop a line of interaction, stop.

## Enforcement

Reports go to the repository owner through GitHub. Responses range from a request to edit a comment, to
closing a thread, to blocking an account, depending on severity.

A decline or a closed thread is usually about **scope** — see [CONTRIBUTING.md](CONTRIBUTING.md) — and
should not be read as a verdict on your view. If a maintainer closes something you raised in good faith,
you are welcome to ask why.

## A note on the subject matter

Some people will find the whole premise objectionable, from either direction: that AI has any place in
biblical study, or that biblical study deserves this much engineering. Both criticisms are reasonable and
both are welcome, stated plainly and once. What is not welcome is repeating them at every contributor
who shows up.
